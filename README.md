# بلينكيت كلون — تطبيق تجارة سريعة (React + Vite)

واجهة عربية RTL بأسلوب بلينكيت: هيدر قابل للطي، ثيم مختلف لكل تبويب، سلة، والأسعار بالدينار العراقي.
الصور حالياً إيموجي مؤقتة — استبدلها بصور حقيقية من `src/data/products.js` و`src/data/collections.js`.

## التشغيل محلياً
```bash
npm install
npm run dev        # خادم التطوير على http://localhost:5173
npm run build      # بناء الإنتاج في dist/
npm run lint       # فحص الكود
```

## النشر على Netlify
1. من لوحة Netlify: **Add new site ← Import an existing project ← GitHub** واختر المستودع.
2. الإعدادات تُقرأ تلقائياً من `netlify.toml` (أمر البناء `npm run build` ومجلد النشر `dist`).
3. اضغط **Deploy** — وكل `git push` لاحق ينشر تلقائياً.


## البوابات (لوحات التحكم)
| البوابة | الرابط | كلمة المرور التجريبية |
|---|---|---|
| الإدارة | `/admin` | **1234** |
| التاجر | `/merchant` | لكل متجر كلمته: النخيل **1111** · بيوتي لاند **2222** · تك ستور **3333** (يديرها الأدمن) |
| المندوب | `/courier` | **1357** |

**تحكم الأدمن الكامل:** لوحة إحصائيات، الطلبات (الحالات + تعيين المندوبين)، المنتجات (إضافة/تعديل/حذف/توفّر + **رابط صورة** يحلّ محل الإيموجي)، **المحتوى** (بانرات العروض، البطاقات الثلاثية، «متاجر يحبها الجميع» — إضافة/تحرير/حذف)، **المظهر** (ألوان الهوية كاملة: الهيدر، اللون الأساسي، الأصفر + كل نصوص الموقع: الاسم، الشعار، الترحيب، العنوان، الفوتر)، التجار (مع **كلمة مرور بوابة لكل متجر**)، المندوبون، والإعدادات (شريط العرض، زمن التوصيل، الرسوم، فتح/إغلاق، إعادة الضبط).

**تحكم التاجر:** لوحة متجره، طلباته فقط (قبول ← قيد التجهيز ← جاهز للتوصيل)، ومنتجاته فقط (سعر، توفّر، **رابط صورة**).

**بوابة المندوب:** الطلبات الجاهزة (استلام وانطلاق)، توصيلاته الجارية (تأكيد التسليم)، أرباحه، ومفتاح التوفّر.

## صفحات المتجر (من فيديوهات التطبيق الأصلي)

> **شريط السلة** يعرض صوراً دائرية مصغّرة لأحدث المنتجات المُضافة (كما في بلينكيت)، و**الشريط السفلي** يُخفى تلقائياً داخل الصفحات الكاملة (السلة/الدفع/التتبع) كي لا يحجب أزرارها.

> **شريط السلة** يعرض صوراً دائرية مصغّرة لأحدث المنتجات المُضافة (كما في بلينكيت)، و**الشريط السفلي** يُخفى تلقائياً داخل الصفحات الكاملة (السلة/الدفع/التتبع) كي لا يحجب أزرارها.
تفاصيل المنتج (مواصفات، وصف، سياسة استبدال، مشابهة، اشترى الناس أيضاً) · السلة الكاملة (شحنات متعددة، فاتورة، توصيل مجاني فوق حد، تعليمات التوصيل، بقشيش) · الدفع (محافظ عراقية + نقداً) · اختيار وإضافة العناوين · **تتبّع الطلب الحي** (خريطة + خط حالة يتحدّث مع إجراءات التاجر والمندوب) · طلباتي + إعادة الطلب · البحث الحي · صفحة التصنيف بشريط جانبي وفرز.

> **شريط السلة** يعرض الآن صوراً دائرية مصغّرة لأحدث المنتجات المُضافة (كما في بلينكيت)، والشريط السفلي يُخفى تلقائياً داخل الصفحات الكاملة كي لا يحجب أزرارها.

### دورة الطلب كاملة (جرّبها!)
1. من المتجر `/` أضف منتجات ← «عرض السلة» ← **تأكيد الطلب**.
2. يظهر الطلب فوراً عند **الأدمن** وعند **التاجر** بحالة «جديد».
3. التاجر: قبول ← جاهز للتوصيل. 4. المندوب: استلام ← **تم التوصيل**. 5. أرباحه = أجر التوصيلة **+ بقشيش الزبون**. 6. الزبون يتابع كل ذلك **لحظياً** من شاشة التتبع، وله إلغاء الطلب وهو «جديد».

> **هل أحتاج Supabase/SQL الآن؟ لا.** الموقع يعمل كاملاً بتخزين محلي (`localStorage`) دون أي إعداد. عند رغبتك بقاعدة بيانات حقيقية (مشاركة البيانات بين الأجهزة)، سيُضاف ملف SQL جاهز للصق في Supabase وربطه عبر تعديل `src/store/appStore.js` فقط — بقية الموقع لن تتغير.
